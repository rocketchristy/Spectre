The database design and architecture includes SKU composition, multiplier-based pricing, and normalized entities with referential integrity. NXTCG ECOMAPP's database is hosted on Db2 13 for z/OS.

ECOMAPP is a normalized, SKU-driven data model with centralized pricing logic and strong referential integrity. The catalog supports 4,824 sellable SKUs derived from 104 base product types and 63 style modifiers, designed to be scalable and consistent with updates.

ECOMAPP database components

The ECOMDB01 database is organized into 15 tablespaces, with each table placed in its own space to support flexibility and management. It includes the USERS, TOKENS, SERIES, STYLES, STYLE_MODIFIERS, PRODUCT_TYPES, PRODUCT_VARIANTS, VARIANT_IMAGES, INVENTORY, CARTS, CART_ITEMS, ADDRESSES, ORDERS, ORDER_ADDRESSES, and ORDER_ITEMS tables, covering the core data structures of the system. Indexing is handled through primary key indexes along with business key indexes to maintain efficient and reliable query performance.

Normalization

ECOMAPP NXTCG demonstrates a high level of normalization, with the schema aligning with Third Normal Form (3NF). All relations satisfy the requirements of First and Second Normal Forms, and the use of composite‑key tables ensures that each non‑key attribute is fully dependent on the entire primary key.
